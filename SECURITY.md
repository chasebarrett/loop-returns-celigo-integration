# Security Policy

## Scope

This repository is published as a **portfolio and educational case study** documenting system design, integration patterns, and operational decision-making.

It does **not** contain:
- source code used in production systems
- credentials, API keys, tokens, or secrets
- internal system identifiers or account IDs
- customer data or personally identifiable information (PII)
- proprietary configuration files or vendor contracts

---

## Responsible Disclosure

If you believe that sensitive, confidential, or proprietary information has been included in this repository unintentionally, please report it to the repository owner so it can be reviewed and removed promptly.

---

## Supported Versions

This repository is not a software package and does not publish versioned releases.  
Security updates consist solely of content review and removal of sensitive information if identified.

---

## Reporting a Vulnerability

Because this repository contains documentation only, traditional security vulnerabilities do not apply.

However, please report:
- accidental inclusion of sensitive information
- screenshots or diagrams that expose internal-only data
- anything that could reasonably violate confidentiality obligations

Reports can be made via a private GitHub message to the repository owner.
